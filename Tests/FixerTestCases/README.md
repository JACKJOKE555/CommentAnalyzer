# FixerTestCases 目录说明（全新注释器测试用例体系）

本目录用于存放 XmlDocRoslynTool（注释器/修复器）所有功能测试用例，已与分析器测试用例一一对应，命名规范为 TC_F_XXX，与分析器 TC_A_XXX 对应。

## 目录结构
- 每个测试用例均有独立的 .cs 测试文件，命名如 TC_F_001_TypeNoCommentBlock.cs。
- 所有测试文件均可被注释器测试用例实际覆盖（与分析器测试用例不同）。
- 新增 Templates/ 子文件夹，存放所有测试文件的原始内容（以 .txt 形式保存），用于测试用例运行前自动恢复测试文件。

## 测试文件恢复机制
- 每次运行注释器测试用例前，先将 Templates/ 下对应的 .txt 文件内容覆盖到测试用例 .cs 文件，确保测试环境一致。
- 测试用例运行后，测试文件内容即为修复结果。

## 用例命名与设计
- 命名规范：TC_F_XXX，与分析器 TC_A_XXX 一一对应。
- 每个用例专门覆盖一种诊断类型或修复流程分支，详见解决方案文档。

## 原有用例声明
- 原有 FixerTestCases/ 下的所有测试用例和测试文件已废弃，不再使用。
- 仅以本目录下新表和新用例为权威。
